# go-kit

`go-kit` 是一个面向 Go 项目的通用工具包集合，目标是沉淀可复用的基础能力，
例如认证（`auth`）、加密、编码、ID 生成、时间处理等。

## 设计目标

- 职责清晰：按领域能力拆分包，避免“万能 util”。
- 低耦合：工具包可被不同服务直接复用。
- 可扩展：先建立稳定目录结构，再按需引入第三方依赖。
- 易测试：每个包具备独立测试用例与边界校验。

## 目录规划（初版）

```text
go-kit/
├─ pkg/
│  ├─ auth/          # 认证与授权相关能力（如 token 签发、校验、刷新）
│  ├─ crypto/        # 加密解密、签名验签、哈希等
│  ├─ idgen/         # 唯一 ID 生成能力
│  └─ timekit/       # 时间相关工具（时区、格式化、时间窗）
├─ .gitignore
└─ README.md
```

## auth 模块计划

- Token Provider 抽象：统一签发、解析、校验接口。
- Claims 规范化：定义基础 claims 与扩展 claims 约束。
- 错误模型：统一认证错误码与错误类型。
- 中间件适配：为 Gin 等框架提供轻量接入层（可选）。
